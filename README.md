# Cetli 📝

**Secure two-way remote control for your Windows PC via Google Drive**

Cetli (Hungarian for "sticky note") lets you execute PowerShell commands on your Windows computer remotely — from your phone, tablet, or any device with access to Claude AI. Commands travel through Google Drive as JSON files, and results come back through Google Docs via Apps Script.

No VPN. No port forwarding. No server. Just a Google account and a Python script.

```
Phone (Claude AI) → Zapier → Google Drive → cetli.py → PowerShell
                                                          ↓
Phone (Claude AI) ← Google Docs ← Apps Script ← result
```

## ✨ Features

- **Two-way communication** — Send commands and read back results
- **Two-factor authentication** — Static key + TOTP (Google Authenticator)
- **Timestamp validation** — Rejects commands older than 5 minutes
- **Command blacklist** — 40+ dangerous patterns blocked automatically
- **Backtick stripping** — Defeats PowerShell obfuscation tricks
- **Constrained Language Mode** — Blocks .NET, COM, and Win32 API calls
- **Double-read verification** — Ensures Drive sync is complete before executing
- **Startup grace period** — Time to cancel stale commands on boot
- **Automatic JSON cleanup** — Corrupt command files are deleted, not looped

## 📋 Prerequisites

- **Windows 10/11** with PowerShell
- **Python 3.x**
- **Google Drive for Desktop** — installed and syncing
- **Google Account** — for Drive and Apps Script
- **Claude AI account** (claude.ai) with Zapier integration enabled
- **Google Authenticator** app on your phone (optional, for TOTP 2FA)

## 🚀 Quick Start

### 1. Download and place files

```
C:\Cetli\              ← LOCAL folder, NOT in cloud sync!
├── cetli.py           ← Main script
├── config.json        ← Your configuration (created from config.example.json)
└── setup_totp.py      ← One-time TOTP setup (optional)
```

> ⚠️ **Important:** Place `cetli.py` and `config.json` in a local folder **outside** your Google Drive sync directory. This prevents the config (which contains your secret key) from being uploaded to the cloud.

### 2. Configure

Copy `config.example.json` to `config.json` and edit it:

```json
{
    "drive_path": "G:\\My Drive\\YourFolder",
    "secret_key": "YourSecretKey123",
    "max_age_minutes": 5,
    "polling_seconds": 10,
    "startup_grace_seconds": 30,
    "totp_enabled": false,
    "totp_secret": "",
    "totp_valid_window": 6,
    "report_url": "",
    "report_token": ""
}
```

| Field | Description |
|-------|-------------|
| `drive_path` | Path to your Google Drive folder where `parancs.json` will appear |
| `secret_key` | A passphrase that must be included in every command |
| `max_age_minutes` | Maximum age of a command before it's rejected (default: 5) |
| `polling_seconds` | How often to check for new commands (default: 10) |
| `startup_grace_seconds` | Delay before processing starts (default: 30) |
| `totp_enabled` | Enable Google Authenticator 2FA (see [TOTP Setup](docs/TOTP_SETUP.md)) |
| `totp_secret` | Generated by `setup_totp.py` — don't fill manually |
| `totp_valid_window` | TOTP tolerance in 30-sec intervals (6 = ±3 minutes) |
| `report_url` | Apps Script webhook URL (see [Apps Script Setup](docs/APPS_SCRIPT_SETUP.md)) |
| `report_token` | Auth token for the Apps Script webhook |

### 3. Run

```
python cetli.py
```

You should see the Cetli banner with your configuration. The script is now watching for commands.

### 4. Auto-start on boot (optional)

Create a `.bat` file in your Windows Startup folder:

```
Path: %APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup\cetli.bat
Content: C:\Python3xx\python.exe C:\Cetli\cetli.py
```

Replace `C:\Python3xx` with your actual Python path.

## 📖 How It Works

### Sending a command

1. You tell Claude AI what you want to do on your PC
2. Claude asks for your secret key (and TOTP code, if enabled)
3. Claude creates a `parancs.json` file in your Google Drive folder via Zapier
4. Google Drive for Desktop syncs the file to your PC
5. `cetli.py` detects the file and runs security checks:
   - ✅ Key validation
   - ✅ Timestamp validation
   - ✅ Blacklist check (with backtick stripping)
   - ✅ TOTP verification (if enabled)
6. If all checks pass, the PowerShell command executes in Constrained Language Mode
7. The result is logged to `naplo.txt` and `parancs.json` is deleted

### Reading results back (optional)

8. `cetli.py` sends the result via HTTP POST to your Apps Script webhook
9. Apps Script appends the result to a Google Docs document
10. Claude reads the document using Google Drive integration

For setup instructions, see [Apps Script Setup](docs/APPS_SCRIPT_SETUP.md).

### Command format

```json
{
    "status": "new",
    "command": "Get-Date",
    "key": "YourSecretKey123",
    "totp": "123456",
    "timestamp": "2026-03-05T21:00:00"
}
```

## 🛡️ Security

### Defense layers

| Layer | What it does | Protects against |
|-------|-------------|-----------------|
| **Secret key** | Static passphrase in every command | Unauthorized access |
| **TOTP 2FA** | Time-based one-time password (optional) | Key theft, replay attacks |
| **Timestamp** | Commands expire after 5 minutes | Replay attacks, stale commands |
| **Blacklist** | 40+ dangerous patterns blocked | Destructive commands, data theft |
| **Backtick stripping** | Removes PowerShell obfuscation | Blacklist bypass attempts |
| **Constrained Language Mode** | Blocks .NET/COM/Win32 | Advanced exploitation |
| **Double-read** | Reads file twice before processing | Partial sync execution |
| **Grace period** | 30-sec delay on startup | Stale commands on reboot |

### Known limitations

This is a personal tool, not enterprise security software. Be aware:

- **Blacklist is not foolproof** — Determined attackers with your key could construct commands that bypass pattern matching (e.g., variable-based obfuscation). The blacklist is defense-in-depth, not the primary protection.
- **The key is the real security** — If an attacker has your secret key (and TOTP secret), they can send commands. Protect your `config.json`.
- **CLM can be bypassed** without AppLocker/WDAC — but the blacklist blocks the most common attempts.

### What's blocked (examples)

```
❌ Remove-Item * -Recurse          → File destruction
❌ Format-Volume                    → Disk formatting
❌ Invoke-WebRequest *.exe          → Remote code download
❌ Invoke-Expression                → Dynamic code execution
❌ `I`n`v`o`k`e-`E`x`p`r`e`s`s`i`o`n → Obfuscated version (backticks stripped)
❌ -EncodedCommand                  → Base64 encoded commands
❌ Get-Content C:\Cetli\config.json → Self-protection
```

## 🤖 Claude AI Integration

To use Cetli with Claude, you need to create a **Claude Skill** — a set of instructions that tells Claude how to interact with the system.

See [`examples/SKILL.example.md`](examples/SKILL.example.md) for a template you can customize with your own paths and document IDs.

### Required Claude.ai setup

1. **Zapier integration** — Enable in Claude.ai settings, connect your Google account
2. **Claude Skill** — Add the skill in Claude.ai → Settings → Customize → Skills

## ⚡ Tips

- **Keep it simple** — Use single quotes in PowerShell commands where possible (avoids JSON escape issues)
- **Test first** — Start with harmless commands like `Get-Date` or `hostname`
- **Check the console** — The colored output tells you exactly what's happening
- **TOTP window** — Don't worry about the 30-sec timer; the ±3 minute tolerance handles Drive sync delays
- **Clean up** — Periodically clear the Google Docs result log if it gets long
